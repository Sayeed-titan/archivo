# Archivo User Manual

This is the living user manual for **Archivo** (the Event/Archive Management
platform built for Spellbound Networks and configurable for other
organizations). It is Markdown on purpose: when the product changes, the
person making the change updates the matching page here in the same pull
request — there is no separate "doc team" pass and no build step required to
read it.

> **Maintaining this manual**
> - Each page below maps to one feature area of the app. If you change what a
>   screen does, edit that page's text. If you change what a screen *looks
>   like*, regenerate its screenshot (see below) — don't hand-edit images.
> - Screenshots live in [`images/`](images/) and are all regenerated by one
>   script: [`.playwright/user-manual-screenshots.mjs`](../../.playwright/user-manual-screenshots.mjs).
>   With the dev server running (`npm run dev:all`) and the demo org seeded
>   (`npm run db:seed`), run:
>   ```
>   node .playwright/user-manual-screenshots.mjs
>   ```
>   from the `archivo/` folder. It logs in as the demo Administrator (and
>   briefly as the demo Viewer for the one role-comparison shot), walks every
>   documented screen, and overwrites the PNGs in `docs/user-manual/images/`
>   in place — filenames stay stable, so this manual's `![](images/...)`
>   links never need to change. Add a new `shot(...)` call to that script
>   whenever you add a page to the manual.
> - Screenshots in this manual come from a real, in-use organization
>   ("Spellbound Networks") rather than a bare demo — that's intentional, it
>   shows the app as it's actually used, not an empty state. Don't recapture
>   against production data unless you're sure it's safe to display.

## Roles referenced throughout this manual

Archivo ships four demo roles (an organization can rename or add to these —
see [Roles & permissions](settings/roles.md)):

| Role | Typical use |
|---|---|
| **Administrator** | Full access: settings, roles, all archives, permanent delete |
| **Archive Officer** | Creates/uploads/manages archives day to day |
| **Department User** | Works within their own department's archives |
| **Viewer** | Read-only — can browse and download, not edit |

Screens or actions that only some roles can reach are called out explicitly
on each page.
